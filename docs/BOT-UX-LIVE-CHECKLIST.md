# Bot UX Live Checklist

## Цель
Быстро проверить, что Telegram-бот после UX-рефакторинга:
- не имеет пустых кнопок;
- не плодит лишние сообщения на обычной навигации;
- показывает понятные русские названия;
- держит контекст экрана через breadcrumbs и подсказки;
- корректно разделяет `screen update` и `artifact delivery`.

## Подготовка
1. Развернуть актуальную версию бота.
2. Открыть два Telegram-сеанса:
   - администратор;
   - обычный клиент.
3. Очистить или зафиксировать последнее сообщение в чате, чтобы было видно, когда бот редактирует экран, а когда создаёт новое сообщение.

## Критерии приёмки
- Нажатие на навигационную кнопку обычно редактирует текущий экран, а не создаёт новый.
- Новое сообщение появляется только там, где это оправдано:
  - конфиг;
  - QR;
  - лог;
  - график;
  - архив;
  - массовая модерация;
  - длительная операция с отдельным результатом.
- У каждого верхнеуровневого раздела есть:
  - понятный русский заголовок;
  - короткая контекстная подсказка;
  - блок `Что можно сделать`.
- Карточки `клиент / администратор / устройство / сервер` показывают:
  - breadcrumbs;
  - статус;
  - ключевые поля;
  - понятные кнопки действий.
- Нет кнопок, которые “ничего не делают”.

## Администратор
### Главное меню
1. Открыть меню администратора.
2. Проверить кнопки:
   - `Обзор`
   - `Выполнение`
   - `Маршруты`
   - `Клиенты`
   - `Операции`
   - `Как видит клиент`
3. Убедиться:
   - названия полностью на русском;
   - раздел открывается редактированием текущего сообщения;
   - сверху есть контекстная подсказка.

### Обзор
1. Открыть `Обзор`.
2. Перейти в:
   - `Сводка`
   - `Здоровье системы`
   - `Проверки работы`
   - `Графики`
   - `Логи`
3. Проверить:
   - `Обновить` обновляет тот же экран;
   - ошибки watchdog оформлены как понятный экран ошибки;
   - `Проверка скорости` не ломает навигацию.

### Выполнение
1. Открыть `Выполнение`.
2. Перейти в:
   - `Сводка выполнения`
   - `Серверы`
   - `Пути до серверов`
   - `Автовыбор сервера`
   - `Предпочтения`
   - `Стеки`
3. Проверить:
   - `Автовыбор сервера` даёт экран результата;
   - `Проверить стеки` даёт экран запуска и не теряет контекст;
   - `Список серверов` открывается с breadcrumbs.

### Карточка сервера
1. Из `Серверы` открыть любой backend.
2. Проверить:
   - виден breadcrumb;
   - видны статус, роль, SSH-порт, вес;
   - кнопки `Проверить`, `Сделать активным`, `Drain`, `Удалить` работают.
3. Для `Проверить`:
   - результат открывается в том же экране;
   - нет пустого ответа.
4. Для `Сделать активным` и `Drain`:
   - результат оформлен как `Готово/Что дальше`;
   - возврат ведёт в раздел серверов.

### Локальная сеть
1. Открыть `Локальная сеть`.
2. Если `gateway mode` выключен:
   - должен быть понятный warning screen, а не сырой технический текст.
3. Если `gateway mode` включён:
   - открыть устройства локальной сети и предпочтения;
   - убедиться, что экран обновляется без спама.

### Маршруты
1. Открыть `Маршруты`.
2. Проверить кнопки:
   - `Проверить адрес`
   - `Обучение задержкам`
   - списки VPN/прямого доступа
   - `Пересобрать маршруты`
   - `Обход блокировок`
3. Проверить:
   - списки открываются в текущем сообщении;
   - `Пересобрать маршруты` показывает корректный result screen;
   - длинные списки не теряют кнопку возврата.

### Клиенты
1. Открыть `Клиенты`.
2. Перейти в:
   - `Все клиенты`
   - `Администраторы`
   - `Запросы`
3. Открыть карточку обычного клиента и карточку администратора.
4. Проверить:
   - одинаковый визуальный стиль;
   - breadcrumbs;
   - статусы и лимиты видны сразу;
   - кнопки действий названы понятно.

### Карточка клиента
1. Нажать:
   - `Отключить/Включить`
   - `Сбросить подключения`
   - `Отозвать доступ`
2. Проверить:
   - действия не зависают;
   - destructive actions имеют confirm;
   - результат оформлен как экран результата;
   - бот не кидает пользователя в случайное место меню.

### Операции
1. Открыть `Операции`.
2. Проверить:
   - `Развернуть обновление`
   - `Откатить версию`
   - `Перезапуск сервисов`
   - `Резервная копия`
   - `Полный экспорт`
   - `Перезагрузить сервер`
3. Проверить:
   - confirm screens выглядят одинаково;
   - backup/export объясняют, что файл придёт отдельным сообщением.

## Клиент
### Главное меню
1. Открыть клиентское меню.
2. Проверить разделы:
   - `Устройства`
   - `Подключение`
   - `Маршруты`
   - `Проверить сайт`
   - `Поддержка`
3. Убедиться:
   - главное меню не перегружено;
   - в шапке есть понятное состояние VPN.

### Устройства
1. Открыть `Устройства`.
2. Проверить:
   - есть подсказка `Что можно сделать`;
   - переход в список устройств идёт без лишнего нового сообщения.
3. Открыть карточку устройства.
4. Проверить:
   - breadcrumb;
   - состояние устройства;
   - handshake;
   - количество исключений и маршрутов через сервер.

### Получение конфига
1. Из карточки устройства открыть получение конфига.
2. Проверить:
   - выбор формата открывается в том же сообщении;
   - дальше конфиг, QR и инструкции приходят отдельными сообщениями;
   - это считается нормальным поведением.

### Удаление устройства
1. Нажать `Удалить устройство`.
2. Проверить:
   - confirm screen оформлен нормально;
   - после подтверждения приходит result screen;
   - бот не теряет меню.

### Подключение
1. Открыть `Подключение`.
2. Проверить:
   - `Статус VPN`
   - `Обновить конфиги`
   - `Получить конфиг`
3. Проверить:
   - `Статус VPN` редактирует экран;
   - `Обновить конфиги` не создаёт лишние служебные сообщения, кроме самих конфигов.

### Маршруты
1. Открыть `Маршруты`.
2. Проверить:
   - `Сайты через VPN`
   - `Исключения`
   - `Через сервер`
3. Для `Исключения` и `Через сервер`:
   - список, добавление, удаление работают;
   - обычная навигация идёт через edit, не answer;
   - возврат ведёт в правильный раздел.

### Поддержка
1. Открыть `Поддержка`.
2. Проверить:
   - `Помощь`
   - `Мои запросы`
   - `Сообщить о проблеме`
3. Проверить:
   - help screen понятен обычному пользователю;
   - после отправки проблемы пользователь возвращается в логичный контекст.

## Chat Noise Check
1. В админском и клиентском чате сделать 10-15 обычных переходов по меню.
2. Проверить, сколько новых сообщений появилось.
3. Ожидаемо новыми сообщениями должны идти только:
   - конфиги и QR;
   - бэкапы/экспорты;
   - графики/логи;
   - массовые запросы на модерации;
   - длительные операции с отдельным результатом.

## Regression Check
1. Нажать старые compat-кнопки, если они ещё встречаются.
2. Убедиться, что они ведут в новый раздел, а не в тупик.
3. Проверить `Назад`, `Меню`, `Обновить` на каждом верхнем уровне.

## Что считать багом
- Кнопка не отвечает.
- Кнопка отвечает только `cb.answer()` без видимого результата.
- Экран внезапно создаёт новое сообщение там, где должен был редактироваться.
- Текст смешивает старый английский и новый русский UI.
- Возврат ведёт не в тот раздел.
- После FSM/confirm пользователь теряет контекст.
- Карточка показывает сырой технический текст без нормального оформления.

## Финальное решение
Приёмка считается успешной, если:
- все основные сценарии admin и client пройдены;
- пустых кнопок нет;
- чат не засоряется при обычной навигации;
- артефактные сообщения отделены от экранов;
- интерфейс выглядит как одна система, а не как набор разрозненных handler’ов.
