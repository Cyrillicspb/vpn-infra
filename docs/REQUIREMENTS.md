## Домашний сервер

### Аппаратные требования

| Параметр | Минимум | Рекомендуется | Для 20+ клиентов |
|---------|---------|---------------|-----------------|
| CPU | 2 ядра, x86_64, 1.5 GHz | 4 ядра, 2.5 GHz | 8 ядер, 3+ GHz |
| RAM | 4 GB | 8 GB | 16 GB |
| SSD | 32 GB | 128 GB | 256 GB |
| Сеть | 100 Мбит/с Ethernet | 1 Гбит/с Ethernet | 1 Гбит/с |
| Питание | — | ИБП 650 VA | ИБП 1000 VA |

> **Важно:** Только **Ethernet**. Wi-Fi неприемлем для сервера — нестабильно, высокая задержка.

### Требования к CPU

- Архитектура: **x86_64** (amd64)
  - ARM (Raspberry Pi 4B+) — работает, но медленнее
  - Виртуализация внутри виртуализации (nested) — может не работать DKMS
- AES-NI: желательно (ускорение шифрования WireGuard)
  - Проверить: `grep -m1 aes /proc/cpuinfo` — должно вернуть `aes`
  - Без AES-NI: ~300–500 Мбит/с вместо ~1+ Гбит/с

### Требования к диску

- **Тип:** SSD (не HDD — Docker + SQLite работают плохо на HDD)
- **Минимум 32 GB:**
  - ОС: ~5 GB
  - Docker-образы: ~8 GB
  - Логи (14 дней): ~1 GB
  - Базы РКН + кэши: ~500 MB
  - Бэкапы (30 дней): ~2–5 GB
  - Prometheus данные (30 дней): ~3–10 GB

### Требования к сети

- **Ethernet** (обязательно, не Wi-Fi)
- Статический LAN IP (через DHCP reservation или netplan)
- Нет CGNAT (белый IP на роутере — проверить на WAN-интерфейсе)
- Рекомендуется: симметричный канал (upload = download)

## VPS

### Аппаратные требования

| Параметр | Минимум | Рекомендуется |
|----------|---------|---------------|
| vCPU | 1 | 2 |
| RAM | 2 GB | 4 GB |
| SSD | 20 GB | 40 GB |
| Сеть | 100 Мбит/с | 1 Гбит/с |
| Трафик | 1 TB/мес | Безлимитный |

### Требования к виртуализации

- **Обязательно KVM** (полная виртуализация)
- **Не подходит:** OpenVZ, LXC (нет поддержки ядерных модулей, нет WireGuard)
- Проверить тип виртуализации: `systemd-detect-virt` на VPS

### Требования к ОС на VPS

- Ubuntu 22.04 LTS или 24.04 LTS
- Чистая установка (setup.sh сам настроит всё)

### Требования к геолокации

- Локация за пределами России (US, Европа, Азия)
- Желательно: не под российской юрисдикцией
- Рекомендуемые: Германия, Финляндия, Нидерланды, США

### Порты (должны быть открыты)

| Порт | Протокол | Назначение |
|------|----------|------------|
| 22 | TCP | SSH (или альтернативный порт) |
| 443 | TCP | REALITY + nginx |
| 443 | UDP | Hysteria2 |
| 8444 | TCP | REALITY-gRPC |
| 8443 | TCP | Nginx admin panels |
| 51820 | UDP | Tier-2 WireGuard туннель |

## Роутер

### Обязательные требования

- **Белый (публичный) IP** на WAN интерфейсе
  - Не CGNAT (не `100.64.x.x` диапазон)
  - Не двойной NAT
- Поддержка **Port Forwarding** (переброс UDP-портов)
- Поддержка **DHCP Reservation** (фиксированный IP для домашнего сервера)

### Желательные требования

- Поддержка **DDNS-клиента** (при динамическом IP): DuckDNS, No-IP, Cloudflare
- Регулируемый **UDP NAT timeout** (увеличить до 300+ секунд для WireGuard)
- **UPnP отключён** (безопаснее, port forwarding настроить вручную)

### Проверка белого IP

```bash
# Проверьте WAN IP роутера через его интерфейс
# и сравните с внешним IP:
curl -s https://icanhazip.com

# Если совпадают — белый IP есть
# Если WAN IP роутера = 100.64.x.x — CGNAT
```

## Интернет-соединение

### Домашний интернет

| Параметр | Минимум | Для 5 клиентов | Для 20 клиентов |
|----------|---------|----------------|-----------------|
| **Upload** | 10 Мбит/с | 25 Мбит/с | 100 Мбит/с |
| Download | 50 Мбит/с | 100 Мбит/с | 500 Мбит/с |
| Задержка до VPS | < 200 мс | < 100 мс | < 50 мс |

> **Upload — критический параметр.** Весь исходящий трафик клиентов проходит через
> upload-канал домашнего сервера. Download VPS — достаточно иметь хороший канал VPS.

### Формула максимального числа клиентов

```
max_clients = upload_Mbps ÷ 5

Примеры:
  10 Мбит/с upload → 2 клиента (для HD-видео)
  50 Мбит/с upload → 10 клиентов
  100 Мбит/с upload → 20 клиентов
  500 Мбит/с upload → 100 клиентов
```

Watchdog мониторит утилизацию upload и предупреждает при > 80%.

### Проверка upload

```bash
# На домашнем сервере:
speedtest-cli --simple
# или
curl -s https://raw.githubusercontent.com/sivel/speedtest-cli/master/speedtest.py | python3 -
```

## Расчёт пропускной способности

| Активность | Upload домашнего | CPU домашнего |
|-----------|-----------------|--------------|
| Idle (keepalive) | ~1 Кбит/с | ~0% |
| Просмотр текста | 1–5 Мбит/с | 1–3% |
| HD видео (720p) | 5–8 Мбит/с | 2–5% |
| Full HD видео (1080p) | 8–15 Мбит/с | 3–7% |
| 4K видео | 15–30 Мбит/с | 5–15% |
