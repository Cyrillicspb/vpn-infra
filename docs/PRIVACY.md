# Приватность

## Что логируется

| Компонент | Что пишет в лог |
|-----------|-----------------|
| dnsmasq | НИЧЕГО (log-queries отключён) |
| watchdog | Переключения стеков, ошибки, uptime |
| nftables | НИЧЕГО |
| nginx | Ошибки только |
| Telegram-бот | Команды (без содержимого) |

## Что НЕ логируется

- DNS-запросы клиентов
- Посещённые сайты
- Трафик клиентов

## Хранение данных

SQLite база содержит:
- chat_id клиентов Telegram
- Имена устройств
- IP-адреса в VPN
- Запросы на добавление доменов

## Рекомендации по безопасности

1. Включите 2FA в Telegram
2. Не передавайте .conf файлы через незащищённые каналы
3. Удаляйте неиспользуемые устройства: `/removedevice`
4. Регулярно меняйте ключи: `/rotate-keys`

## Private DNS (DoH/DoT)

Если на устройстве включён Private DNS (DoH/DoT), DNS-запросы обходят dnsmasq.
Это снижает эффективность split tunneling.

Для корректной работы: отключите Private DNS на устройствах клиентов.
