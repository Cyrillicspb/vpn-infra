# Приватность

## Принципы

1. **Минимальный сбор данных** — хранится только то что необходимо для работы
2. **Без логирования DNS** — dnsmasq не пишет в лог что вы запрашивали
3. **Без истории трафика** — watchdog не хранит посещённые адреса
4. **Данные остаются у вас** — всё на вашем оборудовании, никаких сторонних сервисов

---

## Что логируется

| Компонент | Что пишет в лог | Хранится |
|-----------|-----------------|----------|
| **WireGuard** | Нет логов (ядерный модуль) | — |
| **dnsmasq** | Старты/стопы, ошибки конфига | journald, 14 дней |
| **dnsmasq** | DNS-запросы | **НЕ логируются** (`log-queries` отключён) |
| **watchdog** | Failover, переключения стека, API-запросы | `/var/log/vpn-watchdog.log`, 14 дней |
| **watchdog** | IP-адреса посещённых сайтов | **НЕ логируются** |
| **nftables** | Blocked пакеты (только при явном включении) | **Отключено** |
| **nginx** | HTTP-запросы к панелям | `/var/log/nginx/admin_access.log`, 14 дней |
| **Telegram-бот** | Команды (без содержимого личных сообщений) | journald |
| **fail2ban** | Забаненные IP | `/var/log/fail2ban.log` |
| **backup.sh** | Время, размер бэкапа | `/var/log/vpn-backup.log` |

## Что хранится в БД бота (SQLite)

Файл: `/opt/vpn/telegram-bot/data/vpn_bot.db`

| Таблица | Данные |
|---------|--------|
| `clients` | chat_id, имя устройства, протокол (AWG/WG), peer_id, версия конфига, дата регистрации |
| `devices` | ID устройства, имя, IP-адрес в VPN-подсети, публичный ключ |
| `domain_requests` | chat_id, запрошенный домен, статус (ожидает/одобрен/отклонён) |
| `invite_codes` | хэш кода, кто создал, дата создания/использования |
| `excludes` | device_id, подсеть исключения |

**Не хранится:**
- История посещений
- DNS-запросы
- Объём трафика по IP-адресам
- Содержимое трафика

## Что видит ваш провайдер

| Что | Видит ли провайдер |
|-----|--------------------|
| Факт подключения к VPS | Да — зашифрованные UDP/TCP пакеты к IP вашего VPS |
| Протокол (WireGuard/REALITY/etc) | Нет — трафик замаскирован |
| Сайты которые вы посещаете | Нет — зашифровано |
| Объём трафика | Да — сколько байт передано, но не куда |
| IP вашего VPS | Да |

### Как маскируется трафик

| Стек | Что видит провайдер |
|------|---------------------|
| AWG | Случайный UDP-трафик (нет WireGuard сигнатуры) |
| REALITY | TLS-хендшейк к microsoft.com (корректный, неотличимый) |
| gRPC | TLS+gRPC к cdn.jsdelivr.net |
| CDN | TLS к Cloudflare IP (стандартный HTTPS) |
| Hysteria2 | QUIC+Salamander (случайный UDP) |

## Что видит ваш VPS-провайдер

| Что | Видит ли |
|-----|----------|
| Шифрованный WireGuard трафик к VPS | Да (ваш провайдер → VPS) |
| IP-адреса назначений (куда идёт трафик) | Да (из VPS в интернет) |
| Содержимое трафика | Нет — только если сайт не использует HTTPS |
| Ваши DNS-запросы к заблокированным доменам | Нет — DNS проходит через туннель |

## DNS и приватность

```
[Клиент]
    |
    | DNS запрос: youtube.com
    ↓
[dnsmasq, домашний сервер 10.177.1.1]
    |
    | Домен в vpn-domains.conf?
    |    ДА: server=/youtube.com/10.177.2.2 → запрос через Tier-2 туннель к VPS
    |    НЕТ: upstream DNS (8.8.8.8 через eth0)
    ↓
[VPS DNS]
    |
    | Ответ → dnsmasq → клиент
    | + добавить IP в nft set blocked_dynamic
```

**DNS к заблокированным доменам:**
- Проходит через зашифрованный Tier-2 WireGuard туннель
- Ваш провайдер не видит что вы запрашивали `youtube.com`
- VPS-провайдер видит DNS-запрос к youtube.com (от имени VPS)

**DNS к незаблокированным доменам:**
- Идёт через upstream DNS сервер (8.8.8.8 или 1.1.1.1)
- Ваш провайдер видит DNS-запросы к незаблокированным сайтам
- Для максимальной приватности: настройте DoH на домашнем сервере (не входит в проект)

## Данные Grafana/Prometheus

Метрики содержат:
- Системные показатели: CPU, RAM, диск (без личных данных)
- Количество пиров (без имён или IP клиентов)
- RTT, packet loss, скорость туннеля
- Количество failover переключений

Не содержат:
- Список посещённых сайтов
- Объём трафика per-client
- IP-адреса клиентов

## Ограничения приватности

1. **Telegram как канал управления:** Telegram видит команды боту (не ваш интернет-трафик, но факт управления VPN).

2. **VPS-провайдер:** Видит IP-адреса сайтов которые вы посещаете через VPN (без содержимого). Выбирайте доверенного VPS-провайдера в юрисдикции, отличной от вашей.

3. **Незаблокированный трафик:** Идёт напрямую (split tunneling) — ваш провайдер видит посещение незаблокированных сайтов.

4. **Private DNS на клиенте:** Android/iOS могут использовать DoH, обходя наш dnsmasq. Отключите Private DNS на устройстве.

5. **Cloudflare CDN-стек:** При использовании CDN-стека Cloudflare видит метаданные трафика (IP-адрес назначения, SNI, объём). Это публичная CDN-компания.

## Рекомендации

- Включите **двухфакторную аутентификацию** в Telegram (защита от угона аккаунта бота)
- Не пересылайте `.conf` файлы — они содержат приватный ключ
- Регулярно делайте `/rotate-keys` (раз в 3–6 месяцев)
- Отключите **Private DNS** на Android/iOS для корректной работы split tunneling
- Логи старше 14 дней удаляются автоматически (logrotate)
