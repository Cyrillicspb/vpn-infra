### Старый ПК / ноутбук

Подойдёт любой компьютер с:
- CPU x86_64, 4 ядра
- 4+ GB RAM (лучше 8 GB)
- 64 GB SSD (лучше 128 GB+)
- Ethernet-порт (ноутбукам нужен USB-адаптер или они должны стоять на месте)

**Ноутбук-сервер:** рабочее решение если:
- Есть отдельный блок питания (не на аккумуляторе)
- Подключён по Ethernet (USB-Ethernet адаптер)
- Отключена заставка / Sleep режим (`sudo systemctl mask sleep.target suspend.target hibernate.target`)

## Что использовать как сервер

### Лучший вариант: Mini PC

Плюсы:
- Потребляет 10–25W (vs 80–120W для обычного ПК)
- Бесшумный или почти бесшумный
- Компактный, удобно спрятать рядом с роутером
- Встроенная графика — не нужна видеокарта

Минусы:
- Ограниченная апгрейдабельность (обычно 1–2 слота RAM, 1 SSD)

### Raspberry Pi 5 (8 GB)

Плюсы:
- Дёшево (~6k ₽ за само устройство)
- Потребляет 5–15W
- Компактный

Минусы:
- ARM архитектура — некоторые Docker-образы нужно строить для ARM
- AES шифрование без аппаратного ускорения: ~400 Мбит/с вместо 1+ Гбит/с
- Нужны: корпус, блок питания, SSD через USB3/PCIe
- Microcard SD ненадёжна — используйте USB3 SSD или HAT с NVMe

Рекомендуется: Pi 5 + Pimoroni NVMe Base + SSD 128 GB + официальный блок питания.

### Старый ноутбук / desktop

Рабочее решение. Главный минус — потребление 40–120W vs 10–25W mini PC.

При 10 ₽/кВт·ч:
- Mini PC (15W) → ~130 ₽/мес
- Старый ноутбук (40W) → ~290 ₽/мес
- Старый desktop (80W) → ~580 ₽/мес

### Proxmox / виртуализация

Если у вас уже есть Proxmox или TrueNAS — можно запустить Ubuntu 24.04 как VM (KVM, не LXC).

Требования к VM:
- 4 vCPU, 4 GB RAM, 64 GB SSD
- virtio сетевая карта
- Включить kernel modules passthrough

## Роутер

### Требования

**Критически важно:**
- Белый IP на WAN (не CGNAT)
- Port Forwarding UDP 51820 + 51821
- Стабильная работа 24/7

**Желательно:**
- DDNS-клиент (встроенный или через скрипт)
- Настраиваемый UDP NAT timeout (увеличить до 300+ сек)
- Firewall с правилами

### Рекомендуемые роутеры

| Роутер | Цена | DDNS | Заметки |
|--------|------|------|---------|
| **Keenetic Giga/Ultra** | 6–15k ₽ | Да (KeenDNS) | Лучший выбор для дома, отличный интерфейс |
| **TP-Link Archer AX73/AX90** | 5–12k ₽ | Да | Хороший Wi-Fi 6, простой интерфейс |
| **ASUS RT-AX88U** | 10–20k ₽ | Да | Мощный, поддержка OpenWRT |
| **MikroTik hAP ax3** | 5–10k ₽ | Да | Для продвинутых пользователей, RouterOS |
| **GL.iNet Flint 2** | 5–8k ₽ | Да | OpenWRT из коробки |

**Не рекомендуется:** ISP-роутеры (от провайдера) — часто нет настройки NAT timeout и ограниченный port forwarding.

### Настройка UDP NAT timeout

WireGuard использует UDP. При простое (нет трафика) некоторые роутеры закрывают NAT-сессию.
`PersistentKeepalive = 25` в WireGuard конфиге помогает, но лучше увеличить timeout:

**Keenetic:** Системная конфигурация → Таймаут UDP-сессий → 300–600 сек

**TP-Link/ASUS:** Обычно нет настройки — надеяться на PersistentKeepalive.

**MikroTik:**
```
/ip firewall connection tracking set udp-timeout=5m
```

**OpenWRT:**
```bash
uci set firewall.@defaults[0].udp_timeout=600
uci commit && /etc/init.d/firewall restart
```

## ИБП (UPS)

Домашний сервер должен работать при кратких отключениях питания.

### Зачем нужен ИБП

- Защита от некорректного завершения (повреждение файловой системы, WAL SQLite)
- Продолжение работы VPN при кратких отключениях (<10 мин)
- Защита от скачков напряжения

### Расчёт мощности ИБП

```
Потребление = сервер (15–80W) + роутер (10–30W) + коммутатор (5–10W)
Запас: × 1.5

Пример для Mini PC:
  Mini PC: 20W + Роутер: 15W + Switch: 5W = 40W
  С запасом: 40W × 1.5 = 60VA → достаточно 650 VA UPS

Для старого desktop:
  Desktop: 100W + Роутер: 20W = 120W
  С запасом: 120W × 1.5 = 180VA → нужен минимум 1000 VA UPS
```

### Рекомендуемые ИБП

| Модель | Мощность | Автономность | Цена | Для |
|--------|----------|--------------|------|-----|
| **APC Back-UPS 650** | 650 VA / 400W | 10–20 мин | 3–5k ₽ | Mini PC + роутер |
| **APC Back-UPS 1100** | 1100 VA / 660W | 15–30 мин | 6–10k ₽ | Desktop + роутер |
| **Eaton 5E 650** | 650 VA | 10–20 мин | 3–5k ₽ | Mini PC |
| **CyberPower CP1500** | 1500 VA / 900W | 20–40 мин | 8–15k ₽ | Несколько устройств |

**Важно:** Выбирайте ИБП с автоматическим shutdown (USB-кабель к серверу) — при разряде батареи система корректно выключится.

## Сетевая карта и коммутатор

### Встроенная сетевая карта

Большинство современных ПК и mini PC имеют Gigabit Ethernet — этого достаточно.

Проверить:
```bash
ethtool eth0 | grep Speed   # должно быть 1000 Mb/s или 2500 Mb/s
```

### USB-Ethernet адаптер (для ноутбуков без LAN)

- Рекомендуется: чипсет RTL8153 (USB 3.0, Gigabit) — ~500–1500 ₽
- Избегайте: дешёвые USB 2.0 адаптеры с чипсетом AX88772 — ограничены 100 Мбит/с

### Коммутатор

Если сервер находится не рядом с роутером:
- Достаточно дешёвого неуправляемого Gigabit switch (TP-Link TL-SG105, ~800 ₽)
- Кабель: Cat 5e или Cat 6, прямой

## Хранилище

### SSD vs HDD

**SSD обязателен.** Docker, SQLite, watchdog — все имеют случайные операции чтения/записи.
HDD показывает в 10–50 раз хуже IOPS → замедление запуска, работы, бэкапов.

### Рекомендуемые SSD

| Модель | Форм-фактор | Объём | Цена | Надёжность |
|--------|-------------|-------|------|-----------|
| **Samsung 870 EVO** | SATA 2.5" | 120–250 GB | 2–5k ₽ | Отличная |
| **WD Blue** | SATA 2.5" | 250–500 GB | 2–5k ₽ | Хорошая |
| **Samsung 980** | NVMe M.2 | 250 GB | 2–4k ₽ | Отличная |
| **Crucial P3** | NVMe M.2 | 500 GB | 3–5k ₽ | Хорошая |

> **Совет:** Берите с запасом — Prometheus хранит данные 30 дней, логи ротируются каждые 14 дней.
> 128 GB хватит для 5–10 клиентов с запасом на 2–3 года.

### Мониторинг состояния диска

```bash
# SMART состояние:
sudo apt install smartmontools
sudo smartctl -a /dev/sda

# Watchdog предупредит при диске > 85%
```
