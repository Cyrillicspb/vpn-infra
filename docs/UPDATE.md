# Обновление

## Автоматическое обновление

Watchdog проверяет обновления каждый час через git-зеркало на VPS.
При наличии обновления администратор получает уведомление:

```
🔔 Доступно обновление v1.3 → v1.4
[Обновить] [Пропустить] [Подробнее]
```

Security-обновления помечаются: ⚠️

## Ручное обновление

```
/deploy
```

или из консоли сервера:

```bash
bash /opt/vpn/deploy.sh
```

## Процесс обновления

1. Создаётся снапшот текущей версии
2. Применяются миграции конфигурации
3. Обновляются Docker образы
4. Перезапускаются изменённые сервисы
5. Smoke-тесты
6. При провале → автооткат к снапшоту

## Откат

```
/rollback
```

или:

```bash
bash /opt/vpn/deploy.sh --rollback
```

## Обновление ОС

**Никогда не запускайте** `do-release-upgrade` — это сломает DKMS модули.

Для обновления Ubuntu:
1. Сделайте бэкап: `/backup` или `bash /opt/vpn/scripts/backup.sh`
2. Переустановите Ubuntu 24.04
3. Восстановите: `bash restore.sh backup.tar.gz.gpg`

## git-зеркало на VPS

Обновления получаются с VPS-зеркала, а не напрямую из GitHub.
Это обходит возможную блокировку GitHub из России.

Синхронизация зеркала: cron каждые 30 минут.
