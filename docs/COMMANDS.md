# Команды Telegram-бота

Весь интерфейс на **русском языке**. Бот работает в двух режимах:
- **Администратор** — полный доступ ко всем функциям
- **Клиент** — самообслуживание (свои устройства, конфиги, запросы)

---

## Команды администратора

### Мониторинг и статус

| Команда | Описание | Пример |
|---------|----------|--------|
| `/status` | Статус туннеля: активный стек, RTT, пиры, дата ротации | `/status` |
| `/tunnel` | Детальный статус туннелей и всех 4 стеков | `/tunnel` |
| `/ip` | Внешний IP через VPN и прямой IP | `/ip` |
| `/docker` | Статус всех Docker-контейнеров | `/docker` |
| `/clients` | Список зарегистрированных клиентов и их устройств | `/clients` |
| `/speed` | Speedtest через активный туннель (100 KB + 10 MB) | `/speed` |

### Графики

| Команда | Описание |
|---------|----------|
| `/graph` | Общий обзор (system) |
| `/graph tunnel` | RTT, стеки, failover за 6 часов |
| `/graph speed` | Speedtest история |
| `/graph clients` | Активность пиров, трафик |
| `/graph system` | CPU, RAM, диск (домашний + VPS) |
| `/graph tunnel 24h` | Указать период: `1h`, `6h`, `24h`, `7d` |

### Логи

```
/logs <сервис> [кол-во строк]
```

| Пример | Описание |
|--------|----------|
| `/logs watchdog` | Последние 50 строк watchdog |
| `/logs telegram-bot 100` | 100 строк бота |
| `/logs dnsmasq` | dnsmasq (systemd journal) |
| `/logs hysteria2` | hysteria2 |
| `/logs nginx` | nginx (VPS) |

Если логов много — бот отправит их файлом.

### Управление туннелем

| Команда | Описание |
|---------|----------|
| `/switch reality` | Переключить на REALITY стек |
| `/switch grpc` | Переключить на gRPC стек |
| `/switch cdn` | Переключить на CDN (Cloudflare) |
| `/switch hysteria2` | Переключить на Hysteria2 |

### Управление сервисами

```
/restart <сервис>
```

| Пример | Описание |
|--------|----------|
| `/restart watchdog` | Перезапустить watchdog |
| `/restart dnsmasq` | Перезапустить dnsmasq |
| `/restart telegram-bot` | Перезапустить бот (следующее сообщение — уже новый процесс) |
| `/restart xray-client` | Перезапустить xray REALITY |

```
/reboot    — перезагрузить домашний сервер (требует подтверждение "ДА")
/update    — обновить Docker-образы (требует подтверждение)
```

### Обновление и откат

| Команда | Описание |
|---------|----------|
| `/deploy` | Обновить из git (создаёт снимок, smoke-тест, авто-откат при провале) |
| `/rollback` | Откат к последнему снимку |
| `/rollback список` | Список доступных снимков |

### Управление клиентами

```
/invite                     — создать invite-код (одноразовый, TTL 24ч)
/client disable <имя>       — отключить клиента (peer остаётся, не работает)
/client enable <имя>        — включить клиента
/client kick <имя>          — удалить все устройства клиента
/client limit <имя> <число> — установить лимит устройств (по умолчанию 5)
/broadcast <текст>          — отправить сообщение всем клиентам
```

### Управление маршрутами

```
/vpn add <домен>       — добавить домен в VPN (разблокировать для всех клиентов)
/vpn remove <домен>    — убрать домен из VPN
/direct add <домен>    — добавить домен в прямой доступ (обойти VPN)
/direct remove <домен> — убрать из прямого доступа
/list vpn              — список ручных VPN-доменов
/list direct           — список ручных direct-доменов
/check <домен>         — проверить доступность через VPN
/routes update         — принудительно обновить базы РКН (async)
```

### Запросы клиентов

```
/requests              — список ожидающих запросов на добавление/удаление доменов
```
Для каждого запроса — кнопки [Одобрить] [Отклонить].

### VPS (мульти-VPS)

```
/vps list              — список VPS и их статус
/vps add <IP>          — добавить VPS в ротацию
/vps remove <IP>       — удалить VPS
/migrate-vps <IP>      — перенести конфиги на новый VPS
/migrate-vps <IP> --from-backup — восстановить с бэкапа на новый VPS
```

### Безопасность

```
/rotate-keys           — ротация WireGuard ключей (рассылает новые конфиги)
/renew-cert            — обновить клиентский mTLS сертификат (TTL 2 года)
/renew-ca              — обновить CA сертификат (TTL 10 лет, делать редко)
```

### Диагностика

```
/diagnose <устройство>   — полная диагностика конкретного устройства
```

Выводит: статус пира, last handshake, RTT, маршруты, DNS, kill switch тест.

### Меню

```
/menu                  — показать клавиатуру с основными командами
```

## Команды клиента

### Регистрация

```
/start
```

Запускает FSM-регистрацию:
1. Приветствие + запрос invite-кода (если не администратор)
2. Ввод имени устройства (например: `iPhone`)
3. Выбор протокола: `AWG` (рекомендуется) или `WG`
4. Получение `.conf` файла + QR-кода

> Если chat_id уже зарегистрирован → бот сразу показывает ваши устройства.

### Устройства

| Команда | Описание |
|---------|----------|
| `/mydevices` | Список ваших устройств и их статус |
| `/myconfig <имя>` | Получить конфиг конкретного устройства заново |
| `/adddevice` | Запрос на добавление нового устройства (требует одобрение админа) |
| `/removedevice` | Удалить одно из своих устройств |
| `/update` | Получить актуальные конфиги всех устройств (если изменились) |

### Исключения (split tunneling per device)

```
/exclude add <подсеть>     — добавить подсеть в исключения (не через VPN)
/exclude remove <подсеть>  — убрать из исключений
/exclude list              — список исключений для ваших устройств
```

Пример: `/exclude add 192.168.100.0/24` — офисная подсеть идёт напрямую.

### Запросы

```
/request vpn <домен>     — запрос добавить домен в VPN (на модерацию админа)
/request direct <домен>  — запрос добавить домен в прямой доступ
/myrequests              — статус ваших запросов
```

### Статус и помощь

| Команда | Описание |
|---------|----------|
| `/status` | Текущий статус VPN (туннель, активный стек) |
| `/report <описание>` | Сообщить о проблеме администратору |
| `/help` | Список доступных команд |

## Поведение FSM

**Timeout:** Если вы не ввели ответ в течение **10 минут**, FSM сбрасывается.

**Сброс любой командой:** Если в процессе FSM ввести другую команду (например `/help`) — текущий диалог прервётся и выполнится новая команда.

**Invite-код:** При вводе кода он резервируется на 10 минут. Если FSM завершился по таймауту — код освобождается автоматически.

## Поведение бота для незарегистрированных

- `/start` — приветствие + запрос invite-кода
- Любая другая команда — **молчание** (бот игнорирует)

## Алерты от бота

Бот автоматически присылает уведомления администратору при:

| Событие | Приоритет |
|---------|-----------|
| Туннель упал > 1 мин | 🔴 Критично |
| Все стеки недоступны > 5 мин | 🔴 Критично |
| Watchdog не отвечает | 🔴 Критично |
| RTT > 500ms или потери > 5% | 🟡 Предупреждение |
| Шейпинг обнаружен | 🟡 Предупреждение |
| WG peer stale > 180s | 🟡 Предупреждение |
| Контейнер exited/unhealthy | 🟡 Предупреждение |
| Диск > 85% | 🟡 Предупреждение |
| Сертификат mTLS истекает ≤ 14 дней | 🟡 Предупреждение |
| CA сертификат истекает ≤ 30 дней | 🔴 Критично |
| Внешний IP изменился | ℹ️ Инфо |
| DKMS модуль не собран | 🔴 Критично |
| dnsmasq не отвечает | 🔴 Критично |
| Upload-канал > 80% | 🟡 Предупреждение |
| Deploy провалился → авто-откат | 🔴 Критично |
| Обновление доступно (v1.3→v1.4) | ℹ️ Инфо + кнопки [Обновить] [Пропустить] |

Клиентам при недоступности всех стеков: уведомление «VPN временно недоступен».
