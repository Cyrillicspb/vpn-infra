## Требования и совместимость

### У меня CGNAT. Можно ли установить?

**Нет.** CGNAT (Carrier-Grade NAT) означает что ваш провайдер делит один реальный IP на
множество абонентов — внешний мир не может инициировать соединение к вашему роутеру.

**Как проверить CGNAT:** Зайдите в интерфейс роутера → посмотрите IP на WAN-интерфейсе.
Если он начинается с `100.64.x.x` — это CGNAT.

**Решение:**
1. Позвоните провайдеру и попросите «белый IP» (обычно бесплатно или ~100–300 руб/мес)
2. Если провайдер отказывает — смените тариф или провайдера
3. Если у вас CGNAT + роутер с DDNS — это не поможет, CGNAT не позволяет port forward

### У меня двойной NAT (роутер за роутером). Что делать?

**Двойной NAT:** Когда ваш роутер получает RFC1918 адрес (192.168.x.x, 10.x.x.x) вместо реального IP.

**Решение:**
- Перевести внешнее устройство (модем/ONT провайдера) в режим **Bridge Mode**
- Или добавить DMZ-правило на внешнем устройстве → ваш роутер
- Инструкции зависят от устройства провайдера — уточните у техподдержки

### Работает ли без VPS?

**Нет.** VPS — обязательный компонент. Именно через него идёт зашифрованный трафик к заблокированным ресурсам. Без VPS нет смысла в инфраструктуре.

**Минимальный VPS:** 2 GB RAM, 1 vCPU, 40 GB SSD, KVM. Стоимость ~$3–7/мес.
Рекомендуемые провайдеры: Hetzner, Vultr, DigitalOcean, Contabo.

### Какие ОС поддерживаются для домашнего сервера?

Только **Ubuntu Server 24.04.2 LTS**. Другие дистрибутивы не тестировались и не поддерживаются. Используйте чистую установку.

### Можно ли установить на виртуальную машину (VirtualBox, Proxmox)?

Технически — да, но нужно:
- Для AmneziaWG: поддержка DKMS (виртуальное ядро должно иметь заголовки)
- Для WireGuard: модуль ядра `wireguard` должен быть доступен
- Proxmox LXC контейнер: нужны флаги `tun=1` и `net.ipv4.ip_forward=1`
- **Рекомендуется:** установка на bare metal для надёжности

### Работает ли на Raspberry Pi?

Только Pi 4B или Pi 5 с Ubuntu Server 24.04 arm64. Pi 3B — недостаточно RAM.
Предупреждение: низкая скорость AES (~50 Мбит/с без аппаратного ускорения).

### Можно ли использовать за рубежом?

Технически работает, но **неоптимально** — через VPN будет идти трафик к сайтам,
которые и так доступны. Рекомендуется отключить VPN при пребывании за рубежом.

## Установка

### Сколько времени занимает установка?

25–45 минут при хорошем интернете. Большую часть времени занимает скачивание Docker-образов.

### Можно ли прервать установку?

Да. Установщик идемпотентен — при повторном запуске `sudo bash setup.sh` продолжит с места остановки. Состояние хранится в `/opt/vpn/.setup-state`.

### Что происходит с уже установленной системой при повторном запуске setup.sh?

Установщик проверяет каждый шаг: если шаг уже выполнен — пропускает. Это позволяет:
- Восстановить прерванную установку
- Переустановить отдельные компоненты (удалите строку из `.setup-state`)
- Применить изменения после правки `.env`

### Нужно ли знать Linux для использования?

**Для использования** — нет, управление через Telegram-бота. Достаточно уметь подключиться по SSH для первоначальной установки. После установки всё через бота.

**Для обслуживания** — базовые знания полезны для диагностики редких проблем.

## Использование

### Сколько клиентов поддерживается?

Лимит определяется **upload-каналом** домашнего сервера:

```
max_clients ≈ upload_Mbps ÷ 5
```

Примеры:
- 50 Мбит/с upload → ~10 клиентов (при HD-видео)
- 100 Мбит/с upload → ~20 клиентов
- 500 Мбит/с upload → ~100 клиентов

Watchdog уведомит при утилизации upload > 80%.

### Сколько устройств может иметь один клиент?

По умолчанию **5 устройств**. Администратор может изменить лимит: `/client limit Имя 10`.

### Как добавить нового клиента?

1. Отправьте ему invite-код: `/invite` → код действует 24 часа
2. Клиент пишет боту `/start` и вводит код
3. Проходит FSM-регистрацию (имя устройства + протокол)
4. Получает конфиг автоматически

### Что делать если обновили базы провайдера и какой-то сайт перестал работать?

```
/vpn add сайт.ru    — добавить вручную (работает моментально)
```

Сайт добавляется в ручной список, конфиги клиентов обновляются автоматически (debounce 5 мин).

### Как проверить работает ли split tunneling правильно?

```
/check youtube.com    — должен быть доступен через VPN
/check vk.com         — должен быть доступен напрямую (не заблокирован)
/ip                   — показывает два IP: через VPN и прямой
```

### Что значит "AWG" и "WG"? Что выбрать?

- **AWG (AmneziaWG)** — модифицированный WireGuard с рандомизацией заголовков.
  Невидим для DPI как WireGuard. **Рекомендуется для всех клиентов.**
- **WG (WireGuard)** — стандартный WireGuard. Работает везде, но теоретически
  распознаётся DPI по сигнатуре handshake. Используйте если AWG клиент недоступен.

### Можно ли подключиться с нескольких устройств одновременно?

Да. Каждое устройство — отдельный peer с уникальными ключами. Ограничение — только upload-канал.

### Работают ли звонки (VoIP, FaceTime, Zoom) через VPN?

Незаблокированные сервисы (Telegram, Zoom, большинство сервисов) идут **напрямую** без VPN (split tunneling). Только заблокированный трафик идёт через VPN.

### Как часто обновляются базы заблокированных ресурсов?

Автоматически каждую ночь в 03:00 (cron). Источники: antifilter.download, community.antifilter, opencck, zapret-info, RockBlack.

При изменениях watchdog автоматически рассылает обновлённые конфиги клиентам.

### Нужно ли переустанавливать WireGuard-клиент при обновлении конфига?

Нет. Бот пришлёт `.conf` файл — нужно только импортировать его заново в приложение WireGuard/AmneziaWG (заменив старый конфиг).

## Роутер как клиент

### Можно ли подключить роутер как VPN-клиент?

Да. При добавлении устройства в боте выбирайте **«WireGuard Роутер»**. Все устройства за роутером (телефоны, ноутбуки, телевизоры) автоматически получают VPN без настройки на каждом устройстве.

### Чем конфиг для роутера отличается от конфига для телефона?

Конфиг телефона содержит ~12 000 CIDR в AllowedIPs — точный список заблокированных IP. Это нужно для split tunneling на стороне клиента: только трафик к заблокированным ресурсам идёт в туннель.

Конфиг роутера содержит `AllowedIPs = 0.0.0.0/0` — весь трафик уходит в туннель. Split tunneling при этом выполняет **домашний сервер через nftables**: заблокированный трафик → VPS, незаблокированный → eth0 напрямую. Итоговый результат одинаковый.

Практическая причина: большинство роутеров не принимают .conf файлы размером ~190 КБ.

### Почему заблокированный сайт идёт через IP домашнего сервера, а не VPS?

Значит DNS-запрос к этому домену **не прошёл через dnsmasq** на домашнем сервере.

Механизм split tunneling для домен-based блокировок работает так:
1. DNS-запрос → dnsmasq → резолвит через VPS-DNS → добавляет IP в `blocked_dynamic` set
2. Трафик к этому IP → nftables видит его в `blocked_dynamic` → маркирует fwmark 0x1 → уходит через tun → VPS

Если DNS обходит dnsmasq — шаг 1 не происходит → IP не помечается → трафик идёт напрямую через eth0 домашнего сервера.

**На Keenetic** решается удалением DNS-серверов в разделе **Интернет → Фильтры**.
Подробнее: [TROUBLESHOOTING.md — Роутер как VPN-клиент](TROUBLESHOOTING.md#роутер-как-vpn-клиент-keenetic-и-другие)

### Роутер принимает конфиг, но сайты не открываются — что проверить?

1. Включён ли туннель в Keenetic (тумблер активен)
2. Назначены ли устройства/сегмент на туннель (Приоритеты подключений)
3. Удалены ли сторонние DNS в Интернет → Фильтры
4. Работает ли туннель: `/status` в боте → должен показывать активный стек и RTT

### Работает ли kill switch при использовании роутера?

Да. Если туннель упал — nftables на домашнем сервере блокирует трафик к заблокированным ресурсам от WireGuard-подсети (`ip saddr 10.177.3.0/24 ip daddr @blocked_dynamic oifname != "tun*" drop`). Устройства за роутером не смогут обратиться к заблокированным сайтам напрямую.

Незаблокированные сайты продолжают работать через eth0 без изменений.

## Производительность

### Какую скорость ожидать?

Зависит от стека:
- **Hysteria2:** ~80–90% от скорости VPS-канала
- **REALITY:** ~70–80%
- **gRPC:** ~60–70%
- **CDN:** ~30–50% (дополнительный hop через Cloudflare)

Watchdog автоматически выбирает наиболее быстрый работающий стек.

### VPN работает медленно. Что проверить?

1. `/speed` — покажет скорость текущего стека
2. `/graph speed` — история скоростей
3. `/switch hysteria2` — попробовать самый быстрый стек
4. Проверить upload домашнего интернета: `speedtest-cli` на сервере
5. Проверить нагрузку сервера: `/graph system`

### Влияет ли VPN на скорость незаблокированных сайтов?

**Нет.** Незаблокированный трафик идёт напрямую через eth0, минуя VPN-тоннель. Split tunneling работает как на клиенте (AllowedIPs), так и на сервере (nftables).

## Безопасность и приватность

### Кто может видеть мой трафик?

- **Ваш провайдер** видит: шифрованный трафик к VPS, по протоколу который маскируется под HTTPS
- **VPS-провайдер** видит: IP адреса назначений (без содержимого, трафик зашифрован)
- **Никто** не видит содержимое — трафик зашифрован WireGuard

Подробнее: [PRIVACY.md](PRIVACY.md)

### Безопасно ли хранить приватные ключи на домашнем сервере?

Ключи хранятся в `/opt/vpn/.env` (chmod 600) и доступны только root. Бэкап шифруется GPG перед отправкой в Telegram. Если у злоумышленника нет физического доступа к серверу или root-прав — ключи в безопасности.

### Что будет если VPS взломают?

VPS содержит только Xray-сервер. Расшифровать трафик клиентов без их приватных ключей (которые хранятся только у клиентов) невозможно. Нужно: `/rotate-keys` — сгенерирует новые ключи и разошлёт конфиги.

### Логирует ли бот историю запросов?

**Нет.** dnsmasq не логирует DNS-запросы. Watchdog не хранит историю посещений. В SQLite хранятся только: регистрационные данные пиров, invite-коды, запросы на добавление доменов.

## Технические детали

### Почему WireGuard вместо OpenVPN?

WireGuard: ~4000 строк кода vs 500000 у OpenVPN. Встроен в ядро Linux. В 3–5 раз быстрее. Лучше переключение сетей (мобильный → Wi-Fi).

### Почему AmneziaWG а не обычный WireGuard?

Обычный WireGuard имеет уникальную сигнатуру handshake, по которой DPI может его идентифицировать. AmneziaWG рандомизирует заголовки пакетов, делая трафик неотличимым от случайного.

### Что такое REALITY?

REALITY — замена TLS от XRay. Вместо того чтобы иметь собственный TLS-сертификат (который можно занести в списки блокировки), REALITY «одалживает» TLS-хендшейк у легитимного сайта (microsoft.com). DPI видит корректный TLS к Microsoft, но не может расшифровать — только владелец приватного ключа REALITY может это сделать.

### Что будет если провайдер заблокирует VPS по IP?

Watchdog автоматически переключится на другой стек:
1. REALITY (TCP 443) → если заблокирован по IP
2. gRPC (TCP 8444) → если заблокирован TCP 443
3. CDN через Cloudflare → если заблокированы оба прямых подключения

При блокировке CDN-стека нужно менять VPS.

### Что такое kill switch и зачем он нужен?

Kill switch гарантирует что при падении VPN-туннеля трафик к заблокированным сайтам **полностью блокируется** (не утекает через провайдера в открытом виде).

Реализован двойной защитой в nftables:
1. fwmark routing → при упавшем tun → UNREACHABLE
2. forward chain: `oifname != "tun*" drop` перед `ct established accept`

### Что такое Tier-2 туннель?

Отдельный WireGuard-туннель (10.177.2.0/30) между домашним сервером и VPS. Используется для:
- Prometheus scrape домашнего сервера с VPS
- Watchdog heartbeat → VPS
- DNS upstream для заблокированных доменов (dnsmasq → 10.177.2.2)
- SSH-доступ к VPS через туннель при блокировке прямого SSH

### Что делать если GitHub заблокирован для обновлений?

VPS имеет git-зеркало репозитория (cron sync каждый час). Домашний сервер получает обновления через Tier-2 туннель к VPS, а не напрямую из GitHub.
