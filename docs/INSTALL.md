## 1. Предварительные требования

### Оборудование и ОС

**Домашний сервер:**
- ОС: Ubuntu Server 24.04 LTS (другие версии не поддерживаются)
- RAM: минимум 4 ГБ, рекомендуется 8+ ГБ
- Диск: минимум 64 ГБ SSD
- Подключение: Ethernet (Wi-Fi не подходит для постоянной работы сервера)
- Архитектура: x86_64

**VPS:**
- Тип виртуализации: KVM (не OpenVZ)
- ОС: Ubuntu 22.04 или 24.04 LTS
- RAM: минимум 1 ГБ (рекомендуется 2 ГБ)
- vCPU: минимум 1 (рекомендуется 2)
- Диск: минимум 20 ГБ
- Расположение: за пределами России (US, EU, Asia)
- Доступность: SSH-доступ под root (хотя бы временно, для установки)

**Роутер:**
- Реальный (белый) IP-адрес — обязательно. CGNAT не поддерживается.
- Поддержка port forwarding (проброс портов) — есть в любом роутере.

> **Как проверить белый IP**: зайдите на [2ip.ru](https://2ip.ru) с телефона (через мобильный интернет), запишите IP. Затем зайдите с домашней сети. Если IP совпадает с IP вашего роутера — белый IP есть. Если роутер показывает адрес вида 100.64.x.x, 10.x.x.x или 172.16–31.x.x — это CGNAT.

> **Как проверить тип виртуализации VPS**: подключитесь по SSH и выполните `systemd-detect-virt`. Результат `kvm` — подходит. Результат `openvz` или `lxc` — не подходит.

### Аккаунты

**Telegram (обязательно):**
- Нужен Telegram-аккаунт для создания бота и получения уведомлений.

**Cloudflare (опционально):**
- Бесплатный аккаунт на [cloudflare.com](https://cloudflare.com).
- Нужен только для настройки CDN-стека — наиболее устойчивого к блокировкам.
- Без него система работает на трёх оставшихся стеках.

### Что подготовить заранее

Перед запуском установки соберите следующие данные. Скрипт запросит их интерактивно на фазе 0.

| Данные | Где взять | Обязательно |
|---|---|---|
| IP-адрес VPS | Панель управления провайдера | Да |
| SSH-порт VPS | Панель управления; обычно 22 | Да |
| Root-пароль VPS | Выдаётся при заказе | Да |
| Telegram Bot Token | [@BotFather](https://t.me/BotFather) | Да |
| Telegram Admin Chat ID | [@userinfobot](https://t.me/userinfobot) | Да |
| URL Cloudflare Worker | Создаётся в [Cloudflare Dashboard](https://dash.cloudflare.com) | Нет |

Всё остальное (ключи, пароли, UUID) скрипт генерирует сам.

### Как получить Telegram Bot Token

1. Откройте Telegram и найдите бота [@BotFather](https://t.me/BotFather).
2. Отправьте команду `/newbot`.
3. Введите отображаемое имя бота (например: `Мой VPN`).
4. Введите username бота — должен заканчиваться на `bot` (например: `my_home_vpn_bot`).
5. BotFather ответит сообщением с токеном вида: `7123456789:AAHdqTcvCH1vGWJxfSeofSs0K67XXXXXXXXXX`
6. Скопируйте этот токен — он понадобится при установке.

### Как узнать свой Chat ID

**Вариант 1 (проще):**
1. Найдите в Telegram бота [@userinfobot](https://t.me/userinfobot).
2. Нажмите Start или отправьте `/start`.
3. Бот ответит вашим Chat ID — числом вида `123456789`.

**Вариант 2 (через getmyid_bot):**
1. Найдите [@getmyid_bot](https://t.me/getmyid_bot).
2. Нажмите Start.
3. Бот покажет ваш числовой ID.

### Как проверить, что VPS использует KVM (не OpenVZ)

```bash
# Подключитесь к VPS по SSH и выполните:
systemd-detect-virt
# Ожидаемый ответ: kvm
# Если ответ openvz или lxc — VPS не подходит
```

## 2. Запуск установки

### Windows

1. Скачайте файл `installers/windows/install.bat` из репозитория.
2. Дважды щёлкните по файлу — откроется командная строка.
3. Введите IP домашнего сервера и данные для SSH-подключения.
4. Скрипт подключится к серверу по SSH и запустит установку автоматически.

### macOS

1. Скачайте файл `installers/macos/install.command` из репозитория.
2. Откройте Терминал и выполните:
   ```bash
   chmod +x ~/Downloads/install.command
   ~/Downloads/install.command
   ```
3. Скрипт подключится к серверу по SSH и запустит установку.

### Linux / напрямую на сервере

```bash
# Подключитесь к домашнему серверу по SSH или откройте терминал локально
# Скачайте и запустите скрипт:
curl -fsSL https://raw.githubusercontent.com/Cyrillicspb/vpn-infra/master/setup.sh -o setup.sh
sudo bash setup.sh
```

### Идемпотентность

Скрипт сохраняет прогресс в файл `/opt/vpn/.setup-state`. Если установка прервётся (ошибка, перезагрузка), запустите скрипт повторно — он продолжит с шага, на котором остановился, не повторяя уже выполненные шаги. Ключи и пароли, сгенерированные при первом запуске, сохраняются.

## 3. Что происходит в каждой фазе

Установка состоит из 57 шагов и занимает 25–45 минут. Прогресс отображается как «Шаг N/57».

### Фаза 0 — Предварительные условия (шаги 1–8, ~5 мин)

Единственный интерактивный этап. Скрипт запросит данные и выполнит проверки:

- Шаг 1: Проверка ОС (Ubuntu 24.04) и прав (root/sudo)
- Шаг 2: Автоопределение сети — IP домашнего сервера, gateway, сетевой интерфейс
- Шаг 3: Проверка CGNAT и белого IP
- Шаг 4: Ввод данных пользователем (VPS IP, SSH-порт, пароль, Telegram токен, Chat ID, опции)
- Шаг 5: Проверка доступности VPS по SSH
- Шаг 6: Генерация всех секретов (AWG/WG ключи, REALITY x25519 ×2, UUID ×2, пароли)
- Шаг 7: Запись `/opt/vpn/.env` (chmod 600)
- Шаг 8: Финальная проверка перед установкой

**После фазы 0 можно отойти.** Все остальные фазы полностью автоматические.

### Фаза 1 — Домашний сервер (шаги 9–31, ~15 мин)

Настройка всех компонентов домашнего сервера:

- Шаг 9: Обновление системы, установка базовых пакетов
- Шаг 10: Установка Docker и Docker Compose
- Шаг 11: Создание Docker-сети (172.20.0.0/24)
- Шаг 12: Установка AmneziaWG (DKMS), создание wg0 (10.177.1.0/24, UDP 51820)
- Шаг 13: Установка WireGuard, создание wg1 (10.177.3.0/24, UDP 51821)
- Шаг 14: Настройка nftables (таблица inet vpn, kill switch, fwmark routing, rate limiting)
- Шаг 15: Настройка policy routing (таблица vpn=100, marked=200)
- Шаг 16: Установка dnsmasq (split DNS, nftset= для blocked_dynamic)
- Шаг 17: Установка Python 3, venv, зависимостей для watchdog
- Шаг 18: Копирование watchdog.py и плагинов стеков
- Шаг 19: Создание systemd-юнита watchdog, запуск
- Шаг 20: Установка hysteria2 (клиент)
- Шаг 21: Установка tun2socks
- Шаг 22: Настройка systemd-юнитов (порядок загрузки, vpn-routes, vpn-sets-restore)
- Шаг 23: Первоначальная загрузка баз РКН (antifilter, zapret-info) в blocked_static
- Шаг 24: Настройка cron (03:00 update-routes, 04:00 backup, watchdog failsafe)
- Шаг 25: Установка fail2ban
- Шаг 26: Настройка sysctl (отключение IPv6, BBR)
- Шаг 27: Настройка logrotate
- Шаг 28: Настройка journald (SystemMaxUse=500M)
- Шаг 29: Настройка unattended-upgrades (security only)
- Шаг 30: Блокировка автоматических обновлений ядра (apt-mark hold)
- Шаг 31: Запуск всех Docker-контейнеров домашнего сервера

### Фаза 2 — VPS (шаги 32–44, ~10 мин)

Настройка VPS через SSH-подключение с домашнего сервера:

- Шаг 32: Создание пользователя sysadmin на VPS, перенос SSH-ключа
- Шаг 33: Обновление системы VPS
- Шаг 34: Установка Docker на VPS
- Шаг 35: Настройка nftables на VPS (rate limiting TCP/UDP 443)
- Шаг 36: Генерация CA и клиентского сертификата mTLS (4096 bit)
- Шаг 37: Копирование файлов конфигурации на VPS (`/opt/vpn/`)
- Шаг 38: Запись `/opt/vpn/.env` на VPS (chmod 600)
- Шаг 39: Запуск 3x-ui на VPS (network_mode: host)
- Шаг 40: Настройка инбаундов 3x-ui через API (VLESS-XHTTP-jsdelivr :2083, VLESS-XHTTP-microsoft :2087)
- Шаг 41: Запуск остальных контейнеров VPS (nginx, cloudflared, prometheus, grafana, alertmanager, node-exporter)
- Шаг 42: Настройка git-зеркала на VPS (cron sync с GitHub/jsdelivr каждые 30 мин)
- Шаг 43: Установка fail2ban на VPS
- Шаг 44: Настройка VPS healthcheck (cron каждые 5 мин → Telegram)

### Фаза 3 — Связка (шаги 45–49, ~5 мин)

Объединение домашнего сервера и VPS:

- Шаг 45: Создание WireGuard Tier-2 туннеля (wg-tier2, 10.177.2.0/30, порт VPS 51822)
- Шаг 46: Генерация конфига xray-client (SOCKS :1080, стек reality, VPS:2087)
- Шаг 47: Генерация конфига xray-client-2 (SOCKS :1081, стек reality-grpc, VPS:2083)
- Шаг 48: Генерация конфига hysteria2 (стек hysteria2, VPS UDP:443)
- Шаг 49: Перезапуск всех сервисов, первый запуск watchdog с активным стеком

### Фаза 4 — Проверки (шаги 50–56, ~2 мин)

Автоматические smoke-тесты:

- Шаг 50: DNS-тест (dnsmasq резолвит заблокированный домен, IP попадает в blocked_dynamic)
- Шаг 51: Ping VPS через Tier-2 туннель (10.177.2.2)
- Шаг 52: Curl заблокированного сайта через tun-интерфейс активного стека
- Шаг 53: Проверка watchdog HTTP API (`GET /status` → 200 OK)
- Шаг 54: Проверка Telegram — тестовое сообщение боту
- Шаг 55: Проверка policy routing (fwmark → table marked → tun)
- Шаг 56: Проверка DKMS (модуль AmneziaWG собран для текущего ядра)

### Фаза 5 — Ручные шаги (шаг 57)

Скрипт выводит инструкции по:
- Настройке port forwarding на роутере
- Первому запуску Telegram-бота

## 4. Обязательный ручной шаг: Port Forwarding

Это единственное, что нельзя автоматизировать. Нужно пробросить два UDP-порта на IP домашнего сервера.

Скрипт покажет IP домашнего сервера после завершения установки. Обычно это что-то вроде `192.168.1.100`.

### Keenetic

1. Откройте веб-интерфейс: `192.168.1.1` (или ваш адрес роутера).
2. Перейдите в **Сеть** → **Переадресация**.
3. Нажмите **Добавить правило**.
4. Настройте:
   - Описание: `VPN AWG`
   - Протокол: `UDP`
   - Порт: `51820`
   - IP-адрес: `<IP домашнего сервера>`
   - Порт назначения: `51820`
5. Повторите для порта `51821` (описание: `VPN WG`).
6. Сохраните.

### TP-Link (новый интерфейс)

1. Откройте веб-интерфейс роутера.
2. Перейдите в **Advanced** → **NAT Forwarding** → **Virtual Servers**.
3. Нажмите **Add**.
4. Заполните:
   - Service Type: выберите `Custom`
   - External Port: `51820`
   - Internal IP: `<IP домашнего сервера>`
   - Internal Port: `51820`
   - Protocol: `UDP`
5. Нажмите **Save**.
6. Повторите для порта `51821`.

### ASUS

1. Откройте веб-интерфейс (`192.168.1.1` или `router.asus.com`).
2. Перейдите в **WAN** → **Virtual Server / Port Forwarding**.
3. Включите **Enable Port Forwarding**.
4. Добавьте запись:
   - Service Name: `VPN_AWG`
   - Protocol: `UDP`
   - External Port: `51820`
   - Internal IP: `<IP домашнего сервера>`
   - Internal Port: `51820`
5. Нажмите **+** и добавьте аналогичную запись для порта `51821`.
6. Нажмите **Apply**.

### MikroTik (RouterOS)

```
/ip firewall nat
add chain=dstnat protocol=udp dst-port=51820 action=dst-nat \
    to-addresses=<IP сервера> to-ports=51820 comment="VPN AWG"
add chain=dstnat protocol=udp dst-port=51821 action=dst-nat \
    to-addresses=<IP сервера> to-ports=51821 comment="VPN WG"
```

### Универсальная инструкция (любой роутер)

Ищите в интерфейсе роутера раздел с одним из названий: Port Forwarding, Virtual Server, NAT, Переадресация портов, Проброс портов.

Создайте два правила:

| Протокол | Входящий порт | IP назначения | Исходящий порт |
|---|---|---|---|
| UDP | 51820 | `<IP домашнего сервера>` | 51820 |
| UDP | 51821 | `<IP домашнего сервера>` | 51821 |

### Как проверить, что порты пробрасываются

Откройте [2ip.ru/check-port/](https://2ip.ru/check-port/) и проверьте UDP-порты 51820 и 51821. Если статус «открыт» — всё правильно.

Альтернатива — выполните с другого устройства (например, через мобильный интернет):
```bash
nc -u -z -w3 <ваш_внешний_IP> 51820 && echo "открыт" || echo "закрыт"
```

## 5. Первый запуск: бот и первое устройство

### Регистрация администратора

1. Откройте Telegram и найдите своего бота по username, который вы задали при создании.
2. Нажмите **Start** или отправьте команду `/start`.
3. Бот автоматически зарегистрирует вас как администратора (по Chat ID, указанному при установке).
4. Бот покажет приветственное сообщение и доступные команды.

### Добавление первого устройства

```
/adddevice
```

Бот запросит:
1. Имя устройства (например: `iphone` или `laptop`)
2. Протокол: `AWG` (AmneziaWG — рекомендуется) или `WG` (WireGuard)

После подтверждения бот отправит:
- Файл конфигурации `.conf`
- QR-код (если AllowedIPs ≤ 50 записей)
- Предупреждение о необходимости сохранить конфиг в безопасном месте

### Клиентские приложения

| ОС | AmneziaWG | WireGuard |
|---|---|---|
| Android | [Amnezia](https://play.google.com/store/apps/details?id=org.amnezia.vpn) | [WireGuard](https://play.google.com/store/apps/details?id=com.wireguard.android) |
| iOS | [Amnezia](https://apps.apple.com/app/amneziavpn/id1662184704) | [WireGuard](https://apps.apple.com/app/wireguard/id1441195209) |
| Windows | [AmneziaWG](https://github.com/amnezia-vpn/amneziawg-windows-client/releases) | [WireGuard](https://download.wireguard.com/windows-client/) |
| macOS | [Amnezia](https://apps.apple.com/app/amneziavpn/id1662184704) | [WireGuard](https://apps.apple.com/app/wireguard/id1451685025) |
| Linux | [amneziawg-tools](https://github.com/amnezia-vpn/amneziawg-linux-kernel-module) | `apt install wireguard` |

### Установка конфига в приложение

**AmneziaWG / WireGuard (мобильный):**
1. Откройте приложение → нажмите «+».
2. Выберите «Создать из файла или архива» или «Сканировать QR-код».
3. Импортируйте файл `.conf` из Telegram или отсканируйте QR-код.
4. Включите соединение.

**WireGuard (Windows/macOS):**
1. Откройте приложение.
2. Нажмите «Импортировать туннель(и) из файла».
3. Выберите файл `.conf`, скачанный из Telegram.
4. Включите туннель.

## 7. CDN-стек Cloudflare (опционально)

CDN-стек — наиболее устойчивый к блокировкам. Трафик идёт через Cloudflare, заблокировать его без блокировки всего Cloudflare невозможно. Настройка занимает около 10 минут.

### Создание Cloudflare Worker

1. Зайдите на [dash.cloudflare.com](https://dash.cloudflare.com) и авторизуйтесь.
2. В левом меню выберите **Workers & Pages**.
3. Нажмите **Create application** → **Create Worker**.
4. Дайте Worker имя (например: `vpn-proxy`) и нажмите **Deploy**.
5. Нажмите **Edit code** и вставьте следующий код:

```javascript
export default {
  async fetch(request, env) {
    const url = new URL(request.url);
    // Замените YOUR_VPS_IP на реальный IP вашего VPS
    url.hostname = 'YOUR_VPS_IP';
    url.port = '8080';
    url.protocol = 'http:';
    const modifiedRequest = new Request(url.toString(), {
      method: request.method,
      headers: request.headers,
      body: request.body,
    });
    return fetch(modifiedRequest);
  }
};
```

6. Замените `YOUR_VPS_IP` на IP вашего VPS.
7. Нажмите **Save and Deploy**.
8. Скопируйте URL Worker'а — выглядит как `https://vpn-proxy.your-subdomain.workers.dev`.

### Настройка VPS-стороны CDN-стека

На VPS должен быть запущен Xray с инбаундом VLESS+WebSocket на порту 8080 (localhost). Настройка выполняется через 3x-ui:

1. Войдите в панель 3x-ui (адрес и пароль — в `/opt/vpn/.env` на VPS, переменная `XRAY_PANEL_PASSWORD`). По умолчанию панель доступна по адресу `http://<IP_VPS>:2053`.
2. Перейдите в **Inbounds** → **Add Inbound**.
3. Настройте:
   - Protocol: `VLESS`
   - Port: `8080`
   - Network: `WebSocket`
   - Security: `None` (TLS завершается на Cloudflare Worker)
   - Path: `/ws`
4. Добавьте клиента, скопируйте UUID.

> Детальная настройка CDN-стека: [docs/ARCHITECTURE.md](ARCHITECTURE.md).

### Активация в системе

После создания Worker'а укажите его URL в `/opt/vpn/.env`:

```bash
CF_WORKER_URL=https://vpn-proxy.your-subdomain.workers.dev
```

Перезапустите watchdog:
```bash
sudo systemctl restart watchdog
```

Проверьте через бота: `/status` — CDN-стек должен появиться в списке доступных.

## 8. Повторный запуск при ошибках

Если установка прервалась с ошибкой, скрипт покажет:
- Какой шаг не выполнился
- Описание проблемы
- Способ исправить

После исправления запустите скрипт повторно:
```bash
sudo bash setup.sh
```

Он автоматически пропустит уже выполненные шаги и продолжит с места остановки.

### Принудительный повтор шага

Если нужно принудительно повторить конкретный шаг (например, перенастроить инбаунды VPS):
```bash
# Посмотреть состояние шагов
cat /opt/vpn/.setup-state

# Удалить метку конкретного шага (например шаг 40)
sudo sed -i '/step_40/d' /opt/vpn/.setup-state

# Запустить скрипт — он выполнит только шаг 40 и продолжит дальше
sudo bash setup.sh
```

### Полный сброс и переустановка

```bash
# Удалить файл состояния — скрипт начнёт с начала
sudo rm /opt/vpn/.setup-state
sudo bash setup.sh
```

> **Внимание**: при полном сбросе ключи и пароли будут перегенерированы. Все ранее выданные клиентские конфиги перестанут работать.
